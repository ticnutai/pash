# Google Play Store - Data Safety Questionnaire Answers

Use these answers when filling out the Data Safety section in Google Play Console.

## Does your app collect or share any of the required user data types?
**Yes**

## Data Collection

### Email Address
- **Collected**: Yes (when user creates account - optional)
- **Shared**: No
- **Purpose**: Account management, App functionality
- **Required**: No (app works without login)
- **Encrypted in transit**: Yes
- **User can request deletion**: Yes

### Personal Notes/Bookmarks (User-generated content)
- **Collected**: Yes (stored in cloud when logged in)
- **Shared**: No
- **Purpose**: App functionality
- **Required**: No
- **Encrypted in transit**: Yes
- **User can request deletion**: Yes

### App Settings/Preferences
- **Collected**: Yes (display settings, font preferences)
- **Shared**: No
- **Purpose**: App functionality, Personalization
- **Required**: No
- **Encrypted in transit**: Yes
- **User can request deletion**: Yes

## Data NOT Collected
- Location: No
- Phone number: No
- Payment info: No
- Contacts: No
- Photos/Videos: No
- Files: No
- Calendar: No
- SMS: No
- Health/Fitness: No
- Browsing history: No
- Device identifiers: No
- Crash logs: No (unless added later)
- Ad interactions: No
- App performance data: No (unless added later)

## Security Practices
- **Data encrypted in transit**: Yes (HTTPS)
- **Data encrypted at rest**: Yes (Supabase)
- **Users can request data deletion**: Yes
- **Independent security review**: No
